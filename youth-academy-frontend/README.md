# Youth Academy — Frontend (Angular 22)

فرونت اند Angular v22 لمنصة اكتشاف مواهب كرة القدم "Youth Academy"، بيقلّد تصميم الـ `index.html` الأصلي (نفس الألوان الأحمر/الأسود، نفس شاشة الدخول، نفس شكل الكروت)، لكن متوصّل فعليًا بالباك اند الحقيقي (Node/Express + MongoDB) بدل localStorage، ومضاف عليه قسم **تحليل تطور اللاعب لآخر 3 مواسم**.

## التشغيل

```bash
npm install
npm start
```

هيشتغل على `http://localhost:4200`.

⚠️ لازم الباك اند يكون شغال على `http://localhost:3000` في نفس الوقت (`npm run dev` جوه مجلد `youth-academy-backend`). لو حابب تغيّر البورت أو الدومين، عدّل `src/environments/environment.ts`:

```ts
export const environment = {
  production: false,
  apiOrigin: 'http://localhost:3000',
  apiUrl: 'http://localhost:3000/api',
};
```

## البناء للنشر (Build)

```bash
npm run build
```

الناتج هيتحط في `dist/youth-academy-frontend`.

## الفروقات المتعمدة عن الـ HTML الأصلي

الأصلي كان mockup بالكامل client-side (بيانات وهمية + تسجيل دخول في localStorage). عشان يشتغل مع الباك اند الحقيقي كان لازم:

- **تسجيل الدخول بقى بالبريد الإلكتروني مش الـ username** (الباك اند بيبحث بالـ `email`).
- **التسجيل بقى محتاج اسم كامل + بريد إلكتروني** كمان (مطلوبين في الـ User schema).
- قائمة "مراجعة الطلبات" بقت صفحة حقيقية (`/admin/pending`) بدل سلسلة `confirm()`.
- ضفت زرار **حذف لاعب** للأدمن في صفحة البروفايل (بيستخدم endpoint موجود بالفعل في الباك اند: `DELETE /api/players/:id`) وزرار **اعتماد** لو اللاعب لسه pending.
- لسه سايب دروب داون الـ role في التسجيل زي ما هو (فيه خيار admin) بناءً على كلامك إنك هتظبط الموضوع ده بنفسك بعد المعرض.

## قسم "تحليل تطور اللاعب — آخر 3 مواسم" (الجديد)

في صفحة البروفايل، تحت الفيديو، فيه بلوك جديد بيحلل تطور اللاعب اعتمادًا على آخر 3 قيم من مصفوفة `rating` (اللي أصلاً بتمثل 5 مواسم 2022→2026 في الشارت الأساسي):

- بار تشارت بآخر 3 قيم تقييم.
- Badge تلقائي: 📈 تحسن ملحوظ / ⚖ مستقر / 📉 تراجع (حسب فرق التقييم بين أول وآخر موسم في الـ3).
- جملة عربية توضح نسبة التغيّر.

**ملاحظة مهمة:** الباك اند حاليًا مش بيسجّل إحصائيات (أهداف/أسيست) لكل موسم على حدة، بس إجمالي تراكمي. فالتحليل اعتمد على تطور الـ `rating` بس، بدل ما يختلق أرقام أهداف/أسيست وهمية لكل موسم. لو حابب تحليل أدق، محتاج تضيف للباك اند تتبّع للإحصائيات موسم بموسم.

## تعديلات بسيطة عملتها في الباك اند (مرفقة في `backend-patch/`)

عشان فورم "تقديم طلب انضمام" الجديد يشتغل صح مع الباك اند بتاعك، عملت تعديلين بسيطين وواضحين (مش لمسّيت أي حاجة تانية):

1. **`models/player.model.js`**: ضفت `"rejected"` لقائمة enum بتاعة الـ `status` (كانت بس `["pending","approved"]` رغم إن `admin-controller.js` بيحط `"rejected"` فعليًا — كان ممكن يطيح validation لو اتفعّل `runValidators`).
2. **`controllers/player.controller.js`**: الفورم بيبعت `goals` / `assists` / `matches` كحقول مسطحة (multipart form-data)، لكن الموديل مستنيها جوه كائن `stats` متداخل. ضفت تجميع بسيط ليها في `createPlayer` و `updatePlayer` عشان أرقام اللاعب تتسجل فعليًا بدل ما تتجاهل بصمت.
3. **`.env.example`**: ضفت `JWT_SECRET` (كان ناقص).

الملفات المعدّلة موجودة كاملة في مجلد `backend-patch/` جنب الفرونت اند — تقدر تنسخهم فوق نفس الملفات في `youth-academy-backend/`.
